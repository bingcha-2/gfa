# Product

## Register

product

## Users

冰茶AI 的付费用户:主要是中国开发者,日常通过冰茶AI 的桌面客户端把本地的 Codex、Claude Code、Antigravity 等 CLI/AI 编程工具接管到官方模型服务。他们来「用户中心」(account 网页后台)是为了:查看账户与授权状态、续费/购买套餐、绑定卡密、管理设备席位、核对 Token 用量、处理订单与支付、提交工单、邀请返利、下载客户端。多为高频、目标明确的工具型访问。

## Product Purpose

用户中心是冰茶AI 的自助管理后台。让用户一眼看清「我的账户现在怎么样」(授权是否正常、额度还剩多少、设备/订阅/到期),并能自助完成续费、支付、设备与工单等操作,减少客服压力。成功 = 用户进来 3 秒内知道状态、续费/排障路径清晰、且整体观感与官网一致、有质感、不廉价。

## Brand Personality

冰茶AI = 「冰茶」(琥珀茶 + 冰块 + 电路节点)。声音:可靠、克制、技术、有质感。给开发者用,所以精准、专业;但「会员中心」要有高级感与归属感(像一张会员通行证),不是冷冰冰的管理表格。三词:可靠、精致、技术感。

## Design Direction (account 重设计 · 已选定)

方向 ③「会员通行证 / Membership Pass」。账户被具象成一张高级金属会员卡(琥珀-青铜渐变、压印冰茶徽记、芯片、会员号、有效期、设备打孔点、冰晶反光),配编辑式大标题排版与顶部导航(非侧栏)。深浅两套都做,签名元素(会员卡)放总览。复用官网真实令牌(暖白浅 / 冰蓝靛深 / 琥珀主色 / 琥珀辉光 / Geist 字体 / Codex 绿·Claude 紫·Antigravity 靛),与官网不割裂。落地:co-located `apps/web/src/components/account/account.css`(对标 marketing.css),贯穿全部 9 页 + 登录注册鉴权页。

## Anti-references

- 通用 SaaS 后台默认款:暖米色背景 + 橙色 + 「侧栏 + 顶栏 + 一排指标卡 + 卡片堆」的换皮骨架(就是被否掉的廉价感来源)。
- 渐变填充的廉价卡片、conic-gradient 甜甜圈圆环、低对比扁平描边。
- 与官网割裂的另起炉灶配色/字体。

## Design Principles

1. **品牌即功能,不只是装饰** —— 会员卡/液位这类签名元素要承载真实信息(套餐、到期、设备、额度),不是贴图。
2. **与官网同源** —— 复用官网令牌与字体,account 是官网的延续而非另一个产品。
3. **质感来自工艺,不是颜色** —— 阴影层次、字号层级、等宽数字、辉光、留白,而不是堆颜色。
4. **3 秒看清状态** —— 总览第一屏就回答「我现在正常吗、还剩多少、何时到期」。
5. **深浅都是一等公民** —— 两套都打磨到位。

## Accessibility & Inclusion

正文对比 ≥4.5:1(深浅两套都验证);琥珀色小字用 AA 安全的 `--primary-text` 而非纯主色;尊重 `prefers-reduced-motion`;键盘可达、focus 可见;金属卡上的文字用同色系深色而非纯黑/灰。
