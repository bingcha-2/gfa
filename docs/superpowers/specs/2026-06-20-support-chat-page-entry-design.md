# 在线客服独立页与客户端入口设计

## 目标

为现有 AI 客服能力增加一个客户中心独立页面，并在桌面客户端“使用指南”页顶部提供醒目的入口。用户从客户端遇到额度、登录、订阅、客户端配置等问题时，可以直接打开在线客服页面咨询，而不是只能依赖右下角浮窗或工单入口。

## 已确认方向

采用视觉方案 A：在客户端“使用指南”页面副标题下方、搜索框上方放置醒目横幅入口。入口点击后打开客户中心在线客服页：

`https://my.bcai.lol/account/support`

该入口保留 FAQ 的搜索和折叠列表，不遮挡内容，也不改动客户端主导航。

## 范围

- 新增客户中心页面 `/account/support`。
- 复用现有客户中心布局、主题、认证守卫和 AI 客服会话能力。
- 复用现有客服接口：
  - `GET /api/account/support/conversation`
  - `POST /api/account/support/chat`
- 桌面客户端 `FaqPage` 顶部新增“在线客服”横幅入口。
- `PORTAL_URLS` 增加 `support` 链接，指向 `${PORTAL_BASE}/account/support`。
- 登录页支持 `next` 回跳，未登录用户打开 `/account/support` 后，登录成功回到客服页。

## 非目标

- 不改 AI 客服后端能力、提示词、工具集或知识库逻辑。
- 不在本次设计中启用 `SUPPORT_AGENT_ENABLED` 或配置 LLM 密钥。
- 不替换现有右下角客服浮窗。浮窗可以继续存在，后续如需统一入口再单独做。
- 不改官网 FAQ 页面或后台客服分析页面。

## 客户中心页面设计

页面路由为 `/account/support`，位于客户中心主布局下。页面使用完整工作台形态，而不是简单放大浮窗：

- 顶部区域：标题“在线客服”，说明“额度、订阅、登录和客户端问题都可以在这里咨询”。
- 主聊天区域：展示已有会话消息、助手回复、工具调用状态、转人工提示。
- 输入区域：复用现有发送逻辑，支持 SSE 流式回复。
- 侧边辅助区（桌面宽屏）：显示“可咨询的问题”与“转人工说明”，减少用户不知道怎么问的问题。
- 移动端：单列布局，聊天区域优先，辅助说明收起到聊天上方。

当客服 agent 未启用时，页面不显示空白聊天框，而是展示明确状态：

- 标题：“在线客服暂未开放”
- 说明：引导用户前往工单页面或 FAQ。
- 操作：`查看工单`、`查看 FAQ`

## 客户端入口设计

在 `apps/app/frontend/src/pages/FaqPage.tsx` 中，副标题下方、搜索框上方新增横幅：

- 图标：使用 `MessageCircle` 或 `Bot` 类图标。
- 标题：`在线客服`
- 说明：`遇到额度、登录、订阅或客户端问题，直接和 AI 助手聊。`
- 按钮：`立即咨询`
- 行为：调用 `api.openURL(api.PORTAL_URLS.support)`。

视觉要求：

- 比现有底部反馈卡更鲜艳，使用客户端现有主色 token，不引入新品牌色。
- 高度控制在约 64 到 76px，保证不压掉 FAQ 首屏主体。
- 小窗口下文案可换行，按钮保持可点击，不允许文本溢出。
- 保留现有底部“反馈与帮助”卡，避免用户滚到底部时失去人工/账户中心入口。

## 登录回跳

当前客户中心登录成功后固定跳转 `/account`。新增需求需要支持：

`/account/login?next=/account/support`

设计规则：

- `next` 只允许站内 `/account...` 路径，禁止外部 URL，避免开放重定向。
- 未登录访问 `/account/support` 时，服务端布局 redirect 到 `/account/login?next=/account/support`。
- 登录成功后，如果 `next` 合法，则 `router.push(next)`；否则回到 `/account`。
- 注册页本次不强制支持 `next`，用户如需注册后回跳可后续扩展。

## 数据流

1. 用户在桌面客户端打开“使用指南”。
2. 用户点击顶部“在线客服”横幅。
3. Wails 打开系统浏览器到 `https://my.bcai.lol/account/support`。
4. 若用户已登录，客户中心渲染在线客服页并加载最近会话。
5. 若未登录，跳转登录页并携带 `next=/account/support`。
6. 登录成功后回到在线客服页。
7. 在线客服页通过现有 SSE 接口发送和接收消息。

## 错误与边界

- 客服接口失败：聊天区显示网络错误提示，保留输入框可重试。
- Agent 未启用：展示不可用状态和工单入口，而不是隐藏整个页面。
- 无历史会话：展示现有 greeting 文案，引导用户直接提问。
- 客户端离线或无法打开浏览器：沿用现有 `openURL` 行为，不新增本地弹窗逻辑。

## 测试计划

- Web 客户中心：
  - `/account/support` 页面渲染标题、聊天容器和输入区。
  - agent disabled 时渲染不可用状态和工单入口。
  - `LoginForm` 在合法 `next=/account/support` 下登录成功后跳转到该路径。
  - 非法 `next=https://...` 或非 account 路径被忽略，登录后回 `/account`。
- 桌面客户端前端：
  - `FaqPage` 渲染在线客服横幅。
  - 点击横幅调用 `openURL(PORTAL_URLS.support)`。
  - 文案与按钮在窄宽度下不溢出。

## 实现提示

- 可以从现有 `SupportChatWidget` 抽出共享聊天组件，或先新增页面组件复用其主要逻辑。优先选择改动小且测试清晰的路径。
- 客户端入口需要新增 i18n 文案，至少覆盖现有语言文件，非中文语言可先提供准确英文基线或保持简短直译。
- 由于这是 Web 页面和 Wails 前端入口改动，不需要重构 Go 后端。
